Full Width Layout Overview

Full width layouts utilize the entire viewport width without containers, providing maximum space for content.

info Note: This example page is still using the standard container. Full width layouts are typically used for landing pages, hero sections, or image galleries.

Full Width vs Contained Layout

Full Width (.container-fluid)
Full Width - Spans entire viewport width
Contained (.container)
Contained - Max-width based on breakpoint
When to Use Each
Full Width
  • Landing pages
  • Hero sections
  • Image/video backgrounds
  • Full-screen applications
  • Maps
Contained
  • Blog posts
  • Documentation
  • Forms
  • Content-heavy pages
  • Better readability

Breaking Out of Container

Sometimes you need full-width sections within a contained layout.

CSS Technique
.full-width-section {
  width: 100vw;
  position: relative;
  left: 50%;
  right: 50%;
  margin-left: -50vw;
  margin-right: -50vw;
}

/* Alternative using calc */
.full-width-breakout {
  width: 100vw;
  margin-left: calc(-50vw + 50%);
  margin-right: calc(-50vw + 50%);
}

Full Width Layout Best Practices

  • check_circle Strategic Use: Use full width for visual impact, not everywhere
  • check_circle Content Readability: Even in full-width sections, limit text width for readability
  • check_circle Performance: Optimize large background images and videos
  • check_circle Mobile: Ensure full-width sections work well on small screens
  • warning Avoid: Full-width text that extends across very wide screens
  • warning Avoid: Horizontal scrolling due to overflow

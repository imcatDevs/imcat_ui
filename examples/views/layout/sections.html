Page Sections Overview

Semantic HTML5 elements help structure your page content meaningfully for better accessibility and SEO.

info Best Practice: Use semantic elements like <header>, <main>, <section>, <article>, <aside>, and <footer> instead of generic <div> elements.

Basic Page Structure

Typical Page Layout
<header> - Page Header
<nav> - Navigation Menu
<main> - Main Content Area
<footer> - Page Footer
HTML Structure
<body>
  <header>
    <h1>Site Title</h1>
    <nav>...</nav>
  </header>
  
  <main>
    <!-- Main content -->
  </main>
  
  <footer>
    <p>© 2025 Company</p>
  </footer>
</body>

<header> Element

Represents introductory content, typically containing logo, site title, and primary navigation.

Use Cases
  • Page header with logo and navigation
  • Article header with title and metadata
  • Section header with heading and description

<main> Element

Represents the dominant content of the page. Only one <main> element per page.

Example

Welcome to Our Site

This is the main content area where the primary information is displayed.

Important Rules
  • One per page: Only one <main> element should exist
  • Not nested: Cannot be inside <article>, <aside>, <footer>, <header>, or <nav>
  • Skip to main: Good for accessibility (skip navigation)

Complete Page Structure

Full HTML5 Semantic Structure
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <title>Page Title</title>
</head>
<body>
  <header>
    <h1>Site Title</h1>
    <nav>
      <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">About</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <section>
      <h2>Features</h2>
      <article>
        <h3>Feature 1</h3>
        <p>Description...</p>
      </article>
    </section>

    <aside>
      <h3>Related</h3>
      <ul>...</ul>
    </aside>
  </main>

  <footer>
    <p>© 2025 Company</p>
  </footer>
</body>
</html>

Semantic HTML Best Practices

  • check_circle Use semantic elements: Prefer <header> over <div class="header">
  • check_circle One <main> per page: Only one main content area
  • check_circle Proper nesting: Follow HTML5 content model rules
  • check_circle Headings hierarchy: Use h1-h6 in logical order
  • warning Avoid: Using semantic elements just for styling
  • warning Avoid: Skipping semantic elements when appropriate
Accessibility Benefits
  • Screen readers can navigate by landmarks
  • Skip navigation functionality
  • Better document outline
  • Improved keyboard navigation

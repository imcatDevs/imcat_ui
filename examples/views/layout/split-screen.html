Split Screen Overview

Split screen layouts divide the viewport into two or more equal sections, commonly used for comparisons, before/after displays, or dual-content interfaces.

50/50 Split Screen

Left Panel

Equal width split screen layout

  • Product comparison
  • Before/After displays
  • Dual language content

Right Panel

Perfect for balanced content

  • Feature comparisons
  • Side-by-side views
  • Interactive demos
Code Example
.split-screen {
  display: flex;
  height: 400px;
}

.split-screen__panel {
  flex: 1;
  padding: 2rem;
}

70/30 Split Screen

Main Panel (70%)

Larger panel for primary content

Use when one side has more importance than the other.

Sidebar (30%)

Secondary information or navigation

Vertical Split Screen

Top Panel

Header or featured content area

Bottom Panel

Detail or supplementary information

Code Example
.split-screen--vertical {
  flex-direction: column;

Responsive Split Screen

On mobile, split screens should stack vertically

@media (max-width: 768px) {
  .split-screen {
    flex-direction: column;
  }
  
  .split-screen__panel {
    width: 100%;
    min-height: 300px;
  }
}

Common Use Cases

Visual Comparisons
  • Before/After photos
  • Product variants
  • A/B testing results
  • Design iterations
Dual Content
  • Bilingual content
  • Code and preview
  • Documentation and example
  • Video and transcript

Best Practices

  • check_circle Mobile First: Always stack panels vertically on mobile
  • check_circle Equal Importance: Use 50/50 when both sides are equally important
  • check_circle Clear Separation: Use borders or different backgrounds
  • warning Avoid: Too much content that requires scrolling in each panel

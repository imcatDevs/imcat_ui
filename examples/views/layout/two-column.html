Classic Two Column Layout (70/30)

The most common two-column layout with main content area and a sidebar.

Main Content Area (70%)

This is the primary content area where the main information is displayed.

  • Blog posts or articles
  • Product listings
  • Main application interface
  • Dashboard widgets
Sidebar (30%)

Complementary content:

  • Navigation
  • Filters
  • Widgets
  • Ads

Equal Two Column Layout (50/50)

Equal width columns for balanced content display.

Column 1

Equal width column for balanced layouts.

Use cases:

  • Feature comparisons
  • Before/After displays
  • Dual language content
  • Product variations
Column 2

Equal width column for balanced layouts.

Benefits:

  • Visual balance
  • Equal importance
  • Clean symmetry
  • Easy scanning

Reverse Two Column Layout (30/70)

Sidebar on the left, main content on the right.

Left Sidebar (30%)

Navigation or filters on the left:

  • Category menu
  • Filter options
  • Table of contents
  • Quick links
Main Content (70%)

Primary content area with more space.

Common in:

  • E-commerce product listings
  • Documentation sites
  • Blog archives
  • Search results

Code Examples

70/30 Layout
<div class="row">
  <div class="col-lg-8">
    <!-- Main content -->
  </div>
  <div class="col-lg-4">
    <!-- Sidebar -->
  </div>

<div class="row">
  <div class="col-lg-6">
    <!-- Column 1 -->
  </div>
  <div class="col-lg-6">
    <!-- Column 2 -->
  </div>
</div>

Best Practices

  • check_circle Mobile First: Always consider how columns will stack on mobile
  • check_circle Content Priority: Put most important content first (will appear on top when stacked)
  • check_circle Appropriate Width: Use 70/30 for content-heavy pages, 50/50 for equal importance
  • check_circle Consistent Spacing: Use grid gap utilities for consistent column spacing
  • warning Avoid: Too narrow columns that make reading difficult
